Command-line tools are becoming a primary interface for AI agents — not just humans. But most
CLIs are built for a person at a terminal: they prompt, they print prose, they happily mutate
state, and they tell you nothing a machine can parse. An agent driving them flies blind.

These guidelines describe how to build a CLI that an agent can **discover, drive, and trust** —
while staying excellent for the humans who also use it. They distill the patterns that work, name
the antipatterns that don't, and pin each rule to the assumption about LLM capability behind it,
so the standard can evolve as the models do.

**Status:** v0.1 (draft). A living document — see [Evolution & governance](/evolution/).
